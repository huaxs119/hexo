GUI API

包含的类分为三个部分：

  • 组件类 (component class) 创建用户图形界面，JButton,JLabel,JTextField
  • 容器类 (container class) 包含其他组件，JFrame，JPanel
  • 辅助类 (helper class) 支持GUI组件的，Color,Font

辅助类

布局管理器

setLayout() 方法在容器中设置布局管理器

FlowLayout流式布局

最简单的布局管理器，Jpanel容器默认的布局管理器

按照组件添加的顺序，从左到右将组件排列在容器中，当放满一行，就开始新的一行

对齐方式

LEFT 每一行组件都应该是左对齐
RIGHT 每一行组件都应该是右对齐
CENTER 每一行组件都应该是居中的。默认

组件之间的间隔

int getHgap() 获取组件之间以及组件与Container的边之间的水平间隙
int getVgap() 获取组件之间已经主键与Container的边之间的垂直间隙
void setHgap(int hgap) 设置组件之间以及组件与Container的边之间的水平间隙
void setVgap(int fgap) 设置组件之间以及组件与Container的边之间的垂直间隙

水平间隔默认值是5个像素
垂直间隔默认值是5个像素

创建该布局管理器

FlowLayout() 构造一个新的FlowLayout，它是居中对齐的，默认的水平和垂直间隙是5个单位
FlowLayout(int align) 构造一个新的FlowLayout,他具有指定的对齐方式，默认的水平和垂直间隙是5个单位
FlowLayout(int align,int hgap,int vgap) 创建一个新的流布局管理器，他具有指定的对齐方式以及指定的水平和垂直间隙

GridLayout 网络布局

以网格形式管理组件的，组件按照他们的添加的顺序从左到右排列。可以指定网格中的行数和列数

规则
行数和列数可以是0，但是不能两者都为0。如果一个为0另一个不为0，那么不为0的行或列的大小就是固定的，为0的行或者列由布局管理器动态决定。如果行数和列数都不为0，哪儿以行数为依据，所以行数是固定的，布局管理器会动态的计算列数

Java事件监听机制

事件和事件源

事件(event)可以定义为程序发生了某些事情的信号，例如移动鼠标、点击按钮、敲击键盘。

事件类的根类java.util.EventObject
事件对象包含事件相关的属性，可以使用EventObject类中的实例方法getSource获得事件的源对象

用户动作 源对象 触发的事件类型
点击按钮 JButton ActionEvent
文本域按回车 JTextField ActionEvent
窗口打开/关闭/最小化/关闭 Window WindowEvent
单击/双击/移动/鼠标 Component MouseEvent
点击单选框 JradioButton ItemEvent ActionEvent
点击复选框 JcheckBox ItemEvent ActionEvent

监听器

java中，对象表示的每个事件都是由java.util中EventObject类的子类。

MouseEvent 表示鼠标的动作，例如移动光标、单击、双击
KeyEvent 表示键盘上的按键
ActionEvent 表示用户采取的用户界面操作

事件处理

例：当用户点击按钮时，按钮上的信息会发送变化
